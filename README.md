# CSV結合ツール

大規模な交渉履歴CSVファイルに会員番号を付加するPythonツールです。

## 概要

このツールは以下の処理を行います：
1. 「交渉履歴CSV（TBL_NEGOTIATE.csv）」と「案件情報CSV」を管理番号でJOIN
2. 交渉履歴の各レコードに対応する会員番号を付加
3. メモリ効率を考慮したチャンク処理により、数百万行のデータも処理可能

## 特徴

- 🚀 **高速処理**: チャンク単位での処理により大規模データにも対応
- 🔍 **自動エンコーディング検出**: 文字化けを防ぐため、ファイルのエンコーディングを自動検出
- 📊 **進捗表示**: プログレスバーで処理状況をリアルタイム表示
- 📝 **詳細ログ**: 処理ログと統計情報を自動出力
- 💾 **メモリ効率**: 大きなファイルでもメモリ不足を回避

## インストール

```bash
# リポジトリのクローンまたはファイルのダウンロード後
cd csv_merger

# 必要なパッケージのインストール
pip install -r requirements.txt
```

## 使用方法

### 基本的な使い方

1. CSVファイルを`csv_merger`フォルダに配置
   - `TBL_NEGOTIATE.csv`（交渉履歴）
   - `プラザ全案件ContractList_20250801151720.csv`（案件情報）

2. 実行
```bash
cd csv_merger
python csv_merger.py
```

出力ファイル（`merged_output.csv`）は同じ`csv_merger`フォルダ内に作成されます。

### オプション指定

ファイル名が異なる場合は以下のように指定できます：

```bash
python csv_merger.py --negotiate "交渉履歴.csv" \
                     --contract "契約リスト.csv" \
                     --output "結果.csv" \
                     --chunk-size 50000
```

注意：ファイル名のみを指定した場合、`csv_merger`フォルダ内のファイルが自動的に読み込まれます。

### コマンドラインオプション

- `--negotiate`: 交渉履歴CSVファイルパス（デフォルト: TBL_NEGOTIATE.csv）
- `--contract`: 案件情報CSVファイルパス（デフォルト: プラザ全案件ContractList_20250801151720.csv）
- `--output`: 出力ファイルパス（デフォルト: merged_output.csv）
- `--chunk-size`: 一度に処理する行数（デフォルト: 100000）

## 入力ファイルの形式

### 交渉履歴CSV（TBL_NEGOTIATE.csv）
- Column2が「管理番号」
- 数百万行のデータ

### 案件情報CSV
- A列（1列目）が「管理番号」
- B列（2列目）が「会員番号」
- ヘッダー行あり

## 出力ファイル

- 元の交渉履歴CSVの全列を保持
- Column2（管理番号）の右隣に「会員番号」列を追加
- マッチしなかったレコードの会員番号は空欄

## 処理統計

処理完了後、以下の情報が表示・保存されます：
- 総処理件数
- マッチング成功件数・率
- マッチング失敗件数・率
- 処理時間
- 処理速度（件/秒）
- 最大メモリ使用量

## トラブルシューティング

### メモリ不足エラーが発生する場合
```bash
# チャンクサイズを小さくして実行
python csv_merger.py --chunk-size 50000
```

### 文字化けが発生する場合
- ツールは自動的に複数のエンコーディングを試しますが、それでも文字化けする場合は入力ファイルのエンコーディングを確認してください

### 処理が遅い場合
- チャンクサイズを調整してください（大きくすると速くなりますが、メモリ使用量が増えます）

## 注意事項

- 処理前に十分なディスク容量があることを確認してください
- 大規模なファイルの処理には時間がかかります（数百万行で数分〜数十分）
- 処理中は他の重い処理を避けることを推奨します
